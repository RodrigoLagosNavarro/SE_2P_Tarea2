1. La Componente Humana
¿Qué?: Colaboración entre expertos humanos (especialistas) e
ingenieros del conocimiento.
¿Para qué?: Para suministrar el conocimiento básico y trasladarlo a
un lenguaje informático.
¿Cómo?: Los ingenieros "traducen" el saber empírico mediante
entrevistas y modelado lógico.
Ejemplo: Un experto en geología explica a un ingeniero cómo
identificar minerales por su dureza.

2. La Base de Conocimiento

¿Qué?: Depósito de afirmaciones de validez general (reglas, leyes,
probabilidades).
¿Para qué?: Para almacenar el saber estructurado de forma
permanente.
¿Cómo?: Mediante reglas SI-ENTONCES y relaciones bien
definidas entre conceptos.
Ejemplo: Regla: "Si el motor vibra > 50Hz, entonces hay falla de
rodamiento".
3. La Base de Hechos (Memoria de Trabajo)
¿Qué?: Almacén de información particular y datos específicos del
caso actual.
¿Para qué?: Para guardar los datos temporales del usuario
mientras se resuelve el problema.
¿Cómo?: Se alimenta de las respuestas que el usuario da al
sistema durante la consulta.
Ejemplo: "La temperatura actual del paciente es de 38 grados".
4. El Motor de Inferencia
¿Qué?: Algoritmo o "cerebro" que posee la capacidad de
navegación lógica.
¿Para qué?: Para obtener conclusiones cruzando hechos con
conocimiento.
¿Cómo?: Ejecuta estrategias de razonamiento (hacia adelante o
atrás) para conectar premisas con soluciones.
Ejemplo: Al detectar "lluvia", busca en la base y concluye "activar
limpiaparabrisas".
5. El Subsistema de Explicación

¿Qué?: Módulo que justifica el razonamiento y las conclusiones del
sistema.
¿Para qué?: Para que el usuario entienda y confíe en la respuesta
dada, especialmente en medicina.
¿Cómo?: Rastrea los pasos lógicos realizados por el motor y los
muestra en lenguaje natural.
Ejemplo: "Le recomiendo el medicamento X porque usted indicó
alergia al componente Y".
6. La Interfaz de Usuario
¿Qué?: Canal de comunicación bidireccional entre el humano y la
máquina.
¿Para qué?: Para facilitar el intercambio de información de forma
amigable.
¿Cómo?: Mediante formularios, menús desplegables y pantallas de
diálogo.
Ejemplo: Una ventana de chat que pregunta "¿Dónde localiza el
dolor?".
7. El Usuario
¿Qué?: Persona que interactúa con el sistema para resolver un
problema.
¿Para qué?: Recibir asistencia de nivel experto sin necesidad de
tener al especialista presente.
¿Cómo?: Proporcionando datos de entrada y recibiendo
asesoramiento del sistema.
Ejemplo: Un técnico junior usando un sistema experto para reparar
una antena compleja.
8. La Base de Datos

¿Qué?: Almacén de datos históricos o registros de gran volumen.
¿Para qué?: Para dar soporte de información a largo plazo que
complementa el conocimiento.
¿Cómo?: Consultando registros previos para comparar el estado
actual con patrones pasados.
Ejemplo: Comparar la radiografía actual con el historial del paciente
de 2022.
9. El Subsistema de Adquisición de Conocimiento
¿Qué?: Herramienta que filtra y valida el nuevo conocimiento antes
de guardarlo.
¿Para qué?: Asegurar que las reglas nuevas no contradigan a las
viejas.
¿Cómo?: Analiza la consistencia y redundancia de la información
antes de integrarla.
Ejemplo: Detectar si una regla nueva "El cielo es verde" choca con
la existente "El cielo es azul".
10. El Subsistema de Aprendizaje
¿Qué?: Capacidad de mejorar la base de conocimiento mediante la
experiencia.
¿Para qué?: Evolucionar estructuralmente (nuevas reglas) o
paramétricamente (nuevas probabilidades).
¿Cómo?: Ajustando frecuencias de síntomas o descubriendo
nuevas relaciones en los datos.
Ejemplo: Notar que una plaga aparece más en climas húmedos y
ajustar la probabilidad de diagnóstico.
